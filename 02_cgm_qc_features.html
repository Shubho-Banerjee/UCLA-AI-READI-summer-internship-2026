Introduction

The goal of this notebook is to clean Dexcom G6 continuous glucose monitoring (CGM) data and create summaries at both the participant level and participant-day level. These results will be used later to study how sleep and activity relate to glucose patterns.

I first tested the workflow using one participant and then applied the final pipeline to five CGM participants from the AI-READI mini dataset.

For this assignment, I used all valid CGM days available (up to about 10 days). The AGP-style figure shown here is for learning and visualization purposes only. It should not be considered a clinical 14-day AGP report.

2. Create CGM Cleaning Function

Each Dexcom JSON file has a header (participant ID, timezone) and a body (the glucose readings). This function turns one raw file into one clean table. flatten = TRUE collapses nested JSON fields into single dot-named columns (e.g. blood_glucose.value). The if block is a safety net that only runs if dex$body didn’t parse into a clean table already. It forces each reading into its own row and stacks them.

filter(event_type == "EGV") keeps only actual glucose readings (EGV = Estimated Glucose Value), excluding calibration/alert events. transmute() (rather than mutate()) builds a fresh table with only these named columns, dropping every other raw JSON field. as.numeric(blood_glucose.value) converts text to numbers; anything non-numeric silently becomes NA. This is the source of the missing glucose values found later.

tz_choice maps the device’s raw timezone label to a proper IANA timezone name. IANA names (e.g. America/Los_Angeles) carry real daylight-saving rules for that region, so R can correctly switch between standard/daylight time automatically, as a raw label like “pst” alone can’t do that. TRUE ~ "UTC" is the fallback if nothing matches.

The final mutate() (not transmute(), since we want to keep everything built so far) parses the raw timestamp as UTC, converts it to local time, and derives local date/hour and a daylight-saving flag.

clean_cgm_file <- function(file_path){
  
  dex <- fromJSON(
    file_path,
    flatten = TRUE
  )
  
  # Extract Dexcom CGM event table
  cgm_egv <- dex$body
  
  if (is.list(cgm_egv) && !"event_type" %in% names(cgm_egv)) {
    
    cgm_egv <- do.call(
      rbind,
      lapply(cgm_egv, as.data.frame)
    )
    
  }
  
  cgm_egv <- as.data.frame(cgm_egv)
  
  cgm_clean <- cgm_egv %>%
    filter(event_type == "EGV") %>%
    transmute(
      person_id = dex$header$patient_id,
      event_type,
      time_utc = effective_time_frame.time_interval.start_date_time,
      timezone_abbreviation = dex$header$timezone,
      glucose = as.numeric(blood_glucose.value),
      glucose_unit = blood_glucose.unit
    )
  
  tz_choice <- case_when(
    cgm_clean$timezone_abbreviation[1] %in% c("pst","pdt") ~ "America/Los_Angeles",
    cgm_clean$timezone_abbreviation[1] %in% c("mst","mdt") ~ "America/Denver",
    cgm_clean$timezone_abbreviation[1] %in% c("cst","cdt") ~ "America/Chicago",
    cgm_clean$timezone_abbreviation[1] %in% c("est","edt") ~ "America/New_York",
    TRUE ~ "UTC"
  )
  
  cgm_clean %>%
    mutate(
      time_utc = ymd_hms(time_utc, tz="UTC"),
      time_local = with_tz(time_utc, tz_choice),
      date_local = as.Date(time_local),
      hour_local = hour(time_local),
      is_dst = dst(time_local)
    )
  
}

General-purpose duplicate handler, built in case duplicates existed: keeps a row if it’s not duplicated, or if all its duplicate copies agree on glucose value (n_distinct(glucose)==1); disagreeing duplicates get dropped; agreeing duplicates get collapsed to one copy via slice(1). 5. Determine Valid CGM Days

A CGM day is only trustworthy if the sensor captured enough of it. A “valid day” needs at least 202 of the expected 288 five-minute readings (~70% coverage), which is the same completeness standard used in clinical CGM interpretation.

daily_qc <- cgm_final %>%
  group_by(person_id,date_local) %>%
  summarise(
    n_readings=n(),
    percent_complete=n()/288*100,
    valid_day=n_readings>=202,
    .groups="drop"
  )

6. Create Summary Tables

Table 1: CGM Quality Control Summary

Shows how much data existed, and how much survived cleaning, per participant. This describes data quality only and it says nothing about whether someone’s glucose control was good or bad. left_join() attaches each participant’s valid-day count (calculated separately from daily_qc) onto the QC summary, keeping every row from the summary table.

qc_summary <- cgm_all %>%
  group_by(person_id) %>%
  summarise(
    raw_EGV_readings=n(),
    missing_timestamp=sum(is.na(time_utc)),
    missing_glucose=sum(is.na(glucose)),
    duplicate_timestamps=sum(duplicated(time_utc)),
    unusual_values=sum(unusual_glucose),
    cleaned_readings=sum(!unusual_glucose),
    .groups="drop"
  ) %>%
  left_join(
    daily_qc %>%
      group_by(person_id) %>%
      summarise(
        valid_days=sum(valid_day),
        .groups="drop"
      ),
    by="person_id"
  )

Table 2: Participant-Level CGM Features

One row per participant, summarizing their overall glucose pattern across all valid days. cv (coefficient of variation) expresses glucose variability as a percentage of the mean, so variability can be compared across people regardless of average glucose level. Each pct_* line takes the average of a TRUE/FALSE condition (proportion of readings meeting it), converted to a percentage since CGM readings occur ~every 5 minutes, % of readings approximates % of time. 70–180 mg/dL is the standard target range; above-250 counts are also included in above-180; below-54 counts are also included in below-70 intentional overlap.

participant_summary <- cgm_valid %>%
  group_by(person_id) %>%
  summarise(

    cleaned_readings = n(),

    valid_days = n_distinct(date_local),

    mean_glucose = mean(glucose),
    median_glucose = median(glucose),
    sd_glucose = sd(glucose),

    cv = sd_glucose / mean_glucose * 100,

    pct_70_180 = mean(glucose >= 70 & glucose <= 180, na.rm=TRUE) * 100,

    pct_above_180 = mean(glucose > 180, na.rm=TRUE) * 100,

    pct_above_250 = mean(glucose > 250, na.rm=TRUE) * 100,

    pct_below_70 = mean(glucose < 70, na.rm=TRUE) * 100,

    pct_below_54 = mean(glucose < 54, na.rm=TRUE) * 100,

    .groups="drop"

  )

Table 3: Participant-Day CGM Features

One row per participant, per valid local day. The same features as Table 2, but broken down day by day, plus a daytime/nighttime split. glucose[hour_local >= 6] subsets glucose values to daytime hours (6:00 AM–11:59 PM) before averaging; the mirrored condition (< 6) covers nighttime (12:00 AM–5:59 AM). This is a clock-time split, not actual sleep/wake timing. It’ll be replaced once Garmin sleep data is available.

participant_day_summary <- cgm_valid %>%
  group_by(person_id,date_local) %>%
  summarise(

    cleaned_readings = n(),

    percent_complete = n()/288*100,

    mean_glucose = mean(glucose),

    sd_glucose = sd(glucose),

    pct_70_180 = mean(glucose >=70 & glucose <=180)*100,

    pct_above_180 = mean(glucose >180)*100,

    pct_below_70 = mean(glucose <70)*100,

    daytime_readings = sum(hour_local >= 6),

    nighttime_readings = sum(hour_local < 6),

    daytime_mean = mean(glucose[hour_local >= 6]),

    nighttime_mean = mean(glucose[hour_local < 6]),

    daytime_sd = sd(glucose[hour_local >= 6]),

    nighttime_sd = sd(glucose[hour_local < 6]),

    .groups="drop"

  )

Figure 3: AGP-Style Plot

AGP = Ambulatory Glucose Profile, a standard CGM visualization that overlays all valid days by time-of-day (not calendar date) to show a “typical day” pattern. time_of_day = minutes since midnight, letting readings from different days at the same clock time be grouped together. For each time-of-day, calculates the median glucose plus the 25th–75th percentile spread across days.

agp_participant <- unique(cgm_valid$person_id)[1]

agp <- cgm_valid %>%
  filter(person_id == agp_participant) %>%
  mutate(
    time_of_day = hour_local*60 + minute(time_local)
  ) %>%
  group_by(time_of_day) %>%
  summarise(

    median_glucose = median(glucose),

    lower = quantile(glucose,0.25),

    upper = quantile(glucose,0.75),

    .groups="drop"

  )

Ribbon = day-to-day spread (25th–75th percentile); line = median daily pattern; horizontal lines at 70/180 mg/dL = target range boundaries.

ggplot(
  agp,
  aes(x=time_of_day)
) +

  geom_ribbon(
    aes(
      ymin=lower,
      ymax=upper
    ),
    alpha=0.3
  ) +

  geom_line(
    aes(y=median_glucose)
  ) +

  geom_hline(
    yintercept=70
  ) +

  geom_hline(
    yintercept=180
  ) +

  labs(
    title=paste(
      "AGP-Style CGM Profile:",
      agp_participant
    ),
    x="Minutes After Midnight",
    y="Glucose (mg/dL)"
  )

This is an educational AGP-style visualization, not a clinical AGP. Clinical AGP interpretation typically requires 14 days of ≥70%-complete data, and this dataset has up to ~10 days.
